Project Overview

This project involved improving an existing automated database and table system to enhance Warehouse Management System (WMS) workflows. The goal was to reduce operational complexity, minimize human error, and bridge the gap between experienced and new warehouse staff by consolidating multiple inventory tables into a single, intuitive view.

My Role & Contributions

  • Communicated system improvement ideas to a third-party WMS company.
  • Tested newly implemented tables and reported bugs and edge cases.
  • Collaborated with WMS developers on restructuring inventory tables and SQL-based changes.
  • Trained staff on how to use the new table effectively, including workflow tips and best practices.

Technologies & Skills Used

  • SQL (assisted and reviewed database adjustments)
  • Data Analysis
  • Process Improvement
  • Technical Communication
  • Problem Solving
  • Logistics & Warehouse Operations

Key Outcomes

  • Consolidated multiple inventory tables into a single unified view.
  • Reduced onboarding time for new staff and minimized skill gaps.
  • Lowered risk of missed customer items during daily operations.
  • Prevented team collisions where multiple staff grabbed the same inventory.
  • Improved confidence in inventory accuracy through instant refresh validation.
  • Enabled data-driven decision-making for warehouse management.

Consultant Notes (Polished / Portfolio View)

This is a professional summary of my work improving the Replenishment Plan Report:

  • Created a derived column ReplenResidual to determine which entries require replenishment.
  • Entries with ReplenResidual ≤ 0 are automatically hidden; positive numbers are highlighted.
  • Report updates dynamically with inventory changes, including multiple TABLE and SUP locations per item.
  • Resulted in reduced operational errors, faster onboarding, and improved data-driven decision-making.
